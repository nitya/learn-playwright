What is Playwright

Playwright is an open-source Node.JS library (with supporting tools) that enables end-to-end testing for modern web apps. It’s currently at v1.16.

Here are the main benefits from a developer perspective:

  • Supports Cross-browser testing with Chromium, WebKit, Firefox
  • Has Single API to simplify cross-browser test automation.
  • Has Playwright Test as a built-in test runner for quickstarts
  • Has Playwright Library to simplify third-party app/runner integrations
  • Supports Multiple Languages - including JS/TS, Java, Python, .NET

Why Playwright?

Playwright provides ever-green fast, reliable and capable cross-browser testing and automation for modern web apps.

  • Ever-green => get iterative, continuous updates (vs. monolothic large releases) for faster access to new features, bug fixes.
  • Cross-browser = full API coverage for Chromium, WebKit, Firefox, with browser binaries for Linux, Windows and macOS.
  • Mobile testing = use device emulation to test responsive UI against mobile web browsers for form factors (phone, tablet) or modalities (foldables)
  • Fast + Reliable = timeout-free automation (receives browser signals, is event-driven), browser contexts (1 browser, many isolated execution environments), auto-waits for elements (to be ready), resilient string-based element selectors for convenience
  • Capable = powerful automation support support for multiple pages, input events, dark mode, file actions, network interception, web components etc.

I’ll explore these features and try out relevant APIs in future posts.

Try Playwright (In-Browser)

Try Playwright is an interactive browser-based playground where you can execute and share Playwright scripts in your browser to jumpstart your Playwright exploration - backed by an open-source project!

It comes with a set of built-in examples (for .NET, JavaScript, Java, and Python) that showcase capabilities like:

  • Creating and saving a screenshot (of web page)
  • Creating and saving a PDF (of web page)
  • Logging request/response data (on navigating to link)
  • Using device emulation (e.g., to screenshot page)
  • Setting browser context (geolocation, permissions) pre-testing
  • Recording a video (capturing sequence of events)
  • Logging and modifying network requests
  • Crawling a website (extracting content, taking screenshots)
  • End-to-End testing workflow (for a to-do app)

Select an example and click Run to see the relevant output in browser. Then modify or extend the example code to explore more capabilities interactively. I’ll explore these in more detail in future posts, and share more examples to try out!

You may also be able to setup a TryPlaywright env on k3s to test web apps in embedded device contexts (where k3s is a lightweight Kubernetes env for IoT and Edge computing). I have not tried this yet!
